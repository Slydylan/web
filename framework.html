Emergent Programming / Speculative Origin Map

The project is now framed as a browser-runnable origin map: a seed-address substrate, a small set of dimensionless update operators, and a commit loop that lets patterns become reusable programs. The old SI numerology has been removed from the core. Universality starts at c=1.

Hard reset: natural units

c = G = ħ = k_B = 1

No fixed light-speed number, no privileged biological or audio frequency, and no claim that a special SI value is the source of physics. Frequency is represented as dimensionless phase advance per tick: ω = Δθ/Δτ.

1. Address substrate

Everything begins as an addressable seed. A hash does not prove physics; it gives deterministic reproducibility for experiments.

H = BLAKE2b(seed, context)
n_i ∈ {0,...,15}

2. Phase projection

Each address is projected into phase slots and amplitudes.

A_i = n_i/15
θ_i = 2πn_i/15

3. Operators

Operators are small transformations that can be tested, saved, replayed, and compared.

mix(a,b,t)
wrap(θ)
score(candidate)
commit(candidate)

4. Memory commits

A candidate becomes part of the living program only when it improves compression, prediction, or downstream coherence.

accept iff Δscore > threshold
archive rejected candidates with provenance

Credibility rules

For physicists or technical reviewers, terminal output is not evidence. Every script should publish:

1. variables and units
2. formulas
3. input parameters
4. numerical method
5. output values
6. comparison baseline
7. failure cases

The formula ledger page and the --formulas command in iv.py and gravity.py implement the first pass.

Current scientific claim boundary

This is a speculative computational framework. It may become useful as an emergent-programming environment, a visual search substrate, or a compression/provenance experiment. It should not be marketed as a replacement for established physics unless it produces novel predictions with independent measurements.
